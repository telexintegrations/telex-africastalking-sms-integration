Documentation: Testing SMS Notifier Integration (Telex ↔ Africa’s Talking)

Last Updated: [Insert Date]

Objective

This document outlines the steps to test the integration between Telex and Africa’s Talking (AT) via their sandbox environment. The goal is to ensure SMS notifications triggered in Telex are successfully delivered through Africa’s Talking’s API. Now, the processes below were used because using live numbers would be charged, and I requested the Africa’s Talking team provide an alphanumeric for the live account but they are yet to respond. So the test would only work in their provided testing environment.

Prerequisites

  • Sandbox Access:
    • Email: eimuogbo@gmail.com
    • Password: HNG12-AFRICASTALKING
  • Telex Environment: Configured to send outbound messages via API. This is provided in submission form.
  • Test Phone Numbers: At least 2 numbers (sender and recipient) in international format (e.g., +2348123456789, +25475444441). These can be any random numbers as long as they are in the sandbox testing phone and the Telex channel settings; this would work.

Setup Steps

1. Log into Africa’s Talking Sandbox

Go to Africa’s Talking → Login. Use the sandbox credentials above. Navigate to the Dashboard → Sandbox tab.

2. Click on the simulator you see by the sidebar.

3. Input a random phone number.

This works better if it's African and should be in international format as that is what Africa’s Talking could process. E.g., +2347040321896. It does not have to be real.

Testing Workflow

1. Send a Test SMS via Telex

In the settings of the SMS notifier in Telex, input your phone numbers that were in the sandbox, in this format:

+23470***********, +25490*********

Then go send an SMS in this format: /SMS <message>

1. /SMS All teams should gather for a meeting
2. /SMS Your product is ready, kindly pick them up

2. Verify Delivery in Africa’s Talking Sandbox

Go to the Africa’s Talking dashboard → SANDBOX → SIMULATOR. Check if the test message appears.

3. Confirm Recipient Receives SMS

Ensure the recipient’s phone receives the message. Note: In sandbox mode, messages are simulated.

Troubleshooting

Issue Solution
Message not in AT logs Check Telex’s webhook configuration/network.
Invalid phone number format Use international format (e.g., `+2348...`).

Notes

Sandbox Limitations:

  • No real SMS are sent (simulated only).
  • Rate limits may apply (check AT docs).

Production Migration:

  • Replace sandbox credentials with live API keys.
  • Update API endpoints (remove `sandbox.` from URLs).
